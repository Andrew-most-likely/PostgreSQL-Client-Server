Testing authenticated JSON and CSV endpoints with proper access control

Part D: Secure JSON Data Endpoint

Security Implementation:

  • Requires valid JWT authentication token in Authorization header
  • Returns 401 Unauthorized if no token provided
  • Returns 403 Forbidden if token is invalid or expired
  • Logs all access attempts with username and timestamp
  • JSON file stored in server/data/ directory (never exposed publicly)
  • File path validation prevents directory traversal attacks

Part E: Secure CSV Analytics Endpoint

Security Implementation:

  • CSV file processed server-side only (never sent raw to client)
  • Authentication required via JWT token
  • File stored in protected server/data/ directory
  • Analytics computed on backend, only results returned
  • Access logging for security audit trail
  • Prevents direct file download or unauthorized access
